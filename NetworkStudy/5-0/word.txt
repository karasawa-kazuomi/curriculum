
■ワイヤレス
☞　無線、線を使わないやり方
　　無線LAN：電波による無線通信により複数の機器間でデータの送受信を行なう
　　　　　　 構内ネットワーク（LAN：Local Area Network）のこと
	●WLC
	➡　wireless LAN controller
	　　オフィスなど大人数で無線LAN（Wi-Fi）を利用する際に、
	　　複数台の無線LANアクセスポイント（Wi-Fiアクセスポイント）を管理する装置
	　　通信状況を常に監視し、
	　　負荷分散や端末のアクセスポイント間の移動（ハンドオーバー）を行なう

	●集中管理型AP
	➡　アクセスポイント（AP）
	　　無線LANコントローラによって複数のアクセスポイントを一元管理する管理方式
	　　集中管理型の場合、管理者が設定するのは、無線LANコントローラ
	　　会社よりも一般家庭
	
	●自律型AP
	➡　アクセスポイント（AP）
	　　AP単体で動くもの、APごとに設定
	　　自律型の場合、管理者が設定するのはアクセスポイント
	　　会社など複数のAPを利用する場合に管理が楽

	●CSMA/CA
	➡　搬送波感知多重アクセス/衝突回避
	　　継続的に監視し、一定時間以上継続して空いていることが確認できたら信号の送信
	　　待ち時間は最小限の時間に各端末がランダムな時間
	
	●SOHO
	➡　自宅や小さな事務所を拠点に、個人あるいは少人数で運営される小規模な事業
	　　組織の勤務形態の一つ、、在宅や小さなオフィスで活動する個人事業主や小規模企業を指す
	
	●BSS
	➡　Basic Service Set
	　　1つのAPとそのAPの電波内にいる配下の無線LANクライアントで構成されるネットワーク
	
		〇BSSID
		⇒　無線LANにおけるネットワーク識別子の1つ、48ビット
		　　無線LANネットワークのアクセスポイントのMACアドレスと同じもの
		
	●ESS
	➡　Extended Service Set
	　　複数のBSSで構成される無線LANのネットワーク
	    
		〇ESSID
		⇒　無線LANにおけるネットワークの識別子の1つ
		　　最大32文字までの英数字を設定できる。
		　　ESSIDは無線LANを構成する機器（APや無線LAN端末）に設定する必要がある。
		　　また、ESSIDが同じもの同士が通信することができる。
		
	●ローミング
	➡　複数のWi-Fiアクセスポイントがある環境で、
	　　Wi-Fiに接続したパソコンやスマートフォンが移動した際に、
	　　自動的に近くのアクセスポイントに接続が切り替わる仕組みのこと
	
	●CAPWAP
	➡　集中管理型APと無線LANスイッチ間でやりとりされる制御プロトコル
	　　WLCがAPを制御する方法や両者の間でデータを送受信する方式について定めている
	　　APに接続された端末はWLCを介して外部と通信する
	　　APとWLC間の制御情報のやり取りも、端末とWLC間のデータの送受信も暗号化される
	　　UDPの5246番ポートを、データチャネルにUDPの5247番ポートを使用する。
	
■FHRP
☞　ネットワーク上で複数のルータを束ねて一体的に運用するための制御情報を伝送するプロトコル、総称
　　物理的には複数台あるルータを外部からは一台に見せかけることで、
　　デフォルトゲートウェイを冗長化して信頼性を高めることができる。
　　ルータは自らのIPアドレスと共に、
　　共有される仮想IPアドレスおよび仮想MACアドレスが割り当てられ、
　　他の機器からはこのアドレスで認識される

	●HSRP(Cisco社独自)
	➡　ネットワーク上で複数のルータを束ねて一体的に運用するための制御情報を伝送するプロトコルの1つ
	　　2台のルータを外部からは1台に見せかけることで、
	　　デフォルトゲートウェイを冗長化して信頼性を高めることができる
	　　通常はどちらか一方が稼働状態（アクティブルータ）、
	　　もう一方が待機状態（スタンバイルータ）として運用
	　　アクティブルータが機能を停止するとスタンバイルータが数秒以内にこれを自動的に検知し、
	　　処理を引き継いで肩代わりする
	
	●VRRP(標準規格)
	➡　ネットワーク上で複数のルータを束ねて一体的に運用するための制御情報を伝送するプロトコルの1つ
	　　2台のルータを外部からは1台に見せかけることで、
	　　デフォルトゲートウェイを冗長化して信頼性を高めることができる
	　　通常はどちらか一方が稼働状態（マスタールータ）、
	　　もう一方が待機状態（バックアップルータ）として運用
	　　マスタールータが機能を停止するとバックアップルータが数秒以内にこれを自動的に検知し、
	　　処理を引き継いで肩代わりする

	●GLBP(Cisco社独自)
	➡　ネットワーク上で複数のルータを束ねて一体的に運用するための制御情報を伝送するプロトコルの1つ
	　　物理的には複数台あるルータを外部からは一台に見せかけることで、
	　　デフォルトゲートウェイを冗長化して信頼性を高めることができる
	　　GLBPではグループに設定されたルータ群の中からリーダーであるAVG（Active Virtual Gateway）が選出
	　　残りの各ルータ（AVF：Active Virtual Forwarder）にそれぞれ異なる仮想MACアドレスを配布
	　　
	●アクティブルータ
	➡　仮想IP（仮想MAC）アドレスを宛先として転送されてきたトラフィックをルーティングするルータ
	　　メインルータ
	
	●スタンバイルータ
	➡　アクティブルータがダウンした場合に仮想IPアドレス（仮想MACアドレス）の制御を引き継ぐルータ。
	　　予備用のルータ
	
■QoS
☞　ネットワーク上で提供するサービス品質のこと。ネットワーク機器に
　　QoSを実装することで、ある特定の通信を優先して伝送させたり、帯域幅を確保することができる。

	●輻輳管理
	➡　輻輳を制御」する技術、キューイング・スケジューリングの技術 
	　　輻輳はパケットが混雑しちゃっててうまく相手にとどけられないような状態になってしまうこと
	
	●輻輳回避
	➡　ネットワークおよびインターネットワークの共通のボトルネックでの輻輳を予測、
	　　問題となる前に回避することを目的に、ネットワーク トラフィックの負荷をモニタする

		〇テールドロップ
		⇒　ルータやネットワークスイッチなどの中継機器が輻輳を回避するための機能の一つで、
		　　転送を待つ受信パケットの列（キュー）に新規に受け入れるのをやめ、
		　　着信したパケットをそのまま破棄すること。
		
	●帯域制御
	➡　特定のアプリケーションの通信用に帯域を確保しておく手法
	　　設定方法により、重要な通信に帯域の下限値を定める「帯域保証」、
	　　または帯域を占有させたくない通信に上限値を定める「帯域制限」の2つがある。

		〇シェーピング
		⇒　事前に設定したトラフィック量の最大値を超過したパケットについて、
		　　バッファに蓄積し、順番に処理をさせる手法
		　　トラフィックシェーピングとも呼ばれ、
		　　トラフィックの時系列グラフでは、最大値で平らになるイメージ
		
		〇ポリシング
		⇒　事前に設定したトラフィック量の最大値を超過したパケットについて、
		　　バッファへ蓄積せずに破棄する手法
		　　トラフィックポリシングとも呼ばれ、
		　　トラフィックの時系列グラフでは、山形や谷形などギザギザになるイメージ
		
■WAN
☞　Wide Area Network　:　広域情報通信網
　　広い範囲のネットワークのことを指す
　　いわゆるインターネットがWANにあたり、 
　　複数のLANの集合体というイメージ

	●VPN
	➡　Virtual Private Network　：　仮想専用線
	　　送信側、受信側にそれぞれに設置した機器で「カプセル化」と呼ばれる処理を行い、
	　　第三者には見えない仮想的なトンネルを形成して通信する仕組み「トンネリング」
	　　通信時には正規の利用者であることを確認するために認証を行う、
	　　万一トンネルに侵入された場合に備えて通信内容は暗号化して送信する、
	　　などの技術を併せて採用することで、さらに通信の安全性を高めている
	
	●インターネットVPN
	➡　Virtual Private Network
	　　サービスの一種で、インターネット上に暗号化された専用の通信経路を形成し、
	　　仮想的な組織内ネットワークを構築すること。
	
		〇サイト間VPN
		⇒　A拠点とB拠点を接続するVPNのこと
		　　それぞれの拠点にVPNゲートウェイを設置し、インターネット上に安全な通信経路を構築する
		　　拠点間VPNに適したVPNとしては、コストを抑えつつ複数拠点の接続が行えるIPsec-VPNなどがある。
		　　IPsec-VPN：IPパケットを暗号化して安全な通信を行う技術を用いたVPN
		
		〇リモートアクセスVPN
		⇒　あらかじめパソコンやスマートフォンなどのデバイスにインストールしたクライアントアプリを
		　　使用し、拠点に設置したVPNゲートウェイとの経路を確立するVPNのこと
		　　VPNが必要なときにだけ一時的に接続を確立することがリモートアクセスVPNの特徴
		　　リモートアクセスVPNに適したVPNとしては、
		　　SSL技術対応のWebブラウザがあればすぐに利用可能なSSL-VPNなどがある。
		　　SSL-VPN：データをSSL技術で暗号化しデータを送受信する技術を用いたVPN
		
	●IPsec
	➡　Security Architecture for Internet Protocol
	　　暗号化によってパケットの秘匿や改ざん検知を実現するプロトコル
	　　主にインターネットを介して拠点間を接続する、
	　　インターネットVPNを実現するためのプロトコルとして広く利用される
	
	●GRE
	➡　Generic Routing Encapsulation
	　　ルーティングカプセル化
	　　一つのルーティングプロトコルを使用するデータパケットを
	　　別のプロトコルのパケット内でカプセル化するためのプロトコルのこと
	　　GREは、個別のネットワーク間の接続を簡素化するために、
	　　ネットワーク全体で直接ポイントツーポイント接続を設定する方法の1つ
	
	●クラウド
	➡　インターネットなどのネットワーク経由でユーザにサービスを提供する形態のこと
	　　インターネットなどのネットワークに接続されたコンピューター（サーバー）が提供するサービスを、
	　　利用者はネットワーク経由で手元のパソコンやスマートフォンで使う
	
		〇SaaS（サース）
		⇒　Software as a Service
		　　SaaSはソフトウェアをコンピューターに直接インストールするのではなく、
		　　インターネット上で利用できる仕組みを提供
		　　従来のインストール型ソフトウェアと比べると、
		　　SaaSで構築されたソフトウェアはインストールが不要なためデータ容量を抑えられる、
		　　端末の環境に依存せずにソフトウェアを利用できる等のメリットがある
		
		〇PaaS（パース）
		⇒　Platform as a Service
		　　具体的にはソフトウェアやアプリケーションが動作するのに必要なデータベースや実行環境、
		　　その他インフラやOS等が提供されるサービスを指す
		　　
		
		〇IaaS（イァース／アイアース）
		⇒　Infrastructure as a Service
		　　システムの動作に必要な仮想サーバーやハードディスク、
		　　その他セキュリティ関係等のインフラ全般をサービスとして提供する形態を指す
		　　aaSやPaaSよりも自由度が高く、
		　　コンピューターのスペックやOS選定から行えるのがIaaSの大きな特徴

■セキュリティ
☞　暗号や防御のためのソフトウェア、アクセスの制限などを用いて、データやシステム、
　　通信経路などを保護し、機密漏洩や外部からの攻撃、改ざんなどの危険を排除すること全般を指す
　　保護する対象により、
　　「ネットワークセキュリティ」「コンピュータセキュリティ」「情報セキュリティ」など様々な分野がある

	●脆弱性
	➡　コンピュータのOSやソフトウェアにおいて、プログラムの不具合や設計上のミスが原因となって
	　　発生した情報セキュリティ上の欠陥のことを言う。
	　　脆弱性は、セキュリティホールとも呼ばれる
	
	●脅威
	➡　情報セキュリティリスクを顕在化させるもの
	　　脅威には、大きく分けて人に起因するもの、作業環境に起因するものがあります。
	　　人に起因する人為的脅威は、さらに意図的に起こされるものとそうでないものとに分類される
	
	●マルウェア
	➡　Malware
	　　不正かつ有害な動作を行う意図で作成された悪意のあるソフトウェアや悪質なコードの総称
	
		〇ウイルス
		⇒　他のプログラムに寄生して、そのプログラムの動作を妨げたり、ユーザの意図に反する、
		　　有害な作用を及ぼすためのプログラムで、感染機能や自己拡散機能を持つ
		
		〇ワーム
		⇒　独立のファイルで、他のプログラムの動作を妨げたり、ユーザの意図に反する、
		　　有害な作用を及ぼすためのプログラムで、感染機能や自己拡散機能を持つ
		
		〇トロイの木馬
		⇒　ユーザの意図に反し、
		　　攻撃者の意図する動作を侵入先のコンピュータで秘密裏に行うプログラム
		
	●スプーフィング
	➡　サイバー攻撃：なりすまし
	　　他人のアカウントなどを使って、自分とは違う他人になりすまして、犯罪行為をはたらくといったもの
	
		〇DoS攻撃
		⇒　Denial of Service attack／サービス拒否攻撃
		　　アクセスが集中することでサーバがパンクすることを利用し、
		　　悪意を持ってサーバに大量のデータを送りつけるサイバー攻撃
		　　ウェブサイトやサーバーに対して過剰なアクセスやデータを送付するサイバー攻撃
		　　DoS攻撃とは、1台のコンピューターから攻撃をしてくるサイバー攻撃
		
		〇DDoS攻撃
		⇒　Distributed Denial of Service attack／分散型サービス拒否攻撃
		　　DoS攻撃を、対象のウェブサイトやサーバーに対して複数のコンピューターから大量に行うこと
		　　DDoS攻撃を受けると、サーバーやネットワーク機器などに対して大きな負荷がかかるため、
		　　ウェブサイトへのアクセスができなくなったり、ネットワークの遅延が起こったりする
		　　DDoS攻撃は、複数のコンピューターから一斉にサイバー攻撃
		
		〇中間者攻撃
		⇒　Man In The Middle Attack
		　　バケツリレー攻撃、二者間の通信に割り込み、通信内容の盗聴や改ざんをおこなうもの
		　　中間者攻撃が成立した場合、通信内容の盗聴、内容の差し替えなど、さまざまな攻撃が可能になる
		　　DNSにおける中間者攻撃には、クライアントのDNS問い合わせを盗聴し、
		　　本物の応答がクライアントに届く前に偽の応答を送り込むことで
		　　内容を差し替えるといったものがある
		
	●ソーシャルエンジニアリング
	➡　コンピュータシステムにアクセスするために必要な情報（パスワードなど）やその手がかりを、
	　　それを知る本人や周辺者への接触や接近を通じて盗み取る手法の総称
	　　コンピュータウイルスや通信の盗聴のような情報システムに直接介入する攻撃手法を用いず、
	　　物理的に本人の周辺に近づいて、人間の行動や心理に生じる隙を利用して重要な情報を得る手法を指す
	
	●ブルートフォースアタック
	➡　Brute-force attack（ブルートフォースアタック）
	　　パスワードを破るための攻撃手段
	　　「総当たり攻撃」によるパスワードを解読する方法の一つ
	　　方法は、暗号や暗証番号に対して、理論的にあり得るすべてのパターンを入力して突破を試みるというもの
	
	●WPA
	➡　Wi-Fi Protected Access
	　　安全にWi-Fiで通信を行うためのセキュリティ規格として生まれた
	　　Wi-Fiの通信内容を傍受されないように暗号化するセキュリティ規格の1つ
	
		〇WPAパーソナル
		⇒　暗号化の規格がWPAで、事前共有鍵（PSK）を使って認証するやり方
		　　WPAパーソナルでは「WPA」と呼ばれる規格に沿って、暗号化されます
		
		〇WPAエンタープライズ
		⇒　無線LANにおける暗号化とか認証のやり方のひとつ
		　　「WPAという規格に従って暗号化するよ！
		　　あと、認証用のコンピュータを使って認証するよ！」なやり方のこと
		
	●サプリカント
	➡　ネットワーク上の利用者や端末の認証において、
	　　認証を要求する側（クライアント側）の機器やソフトウェアのこと
	　　また、クライアント側で認証のやり取りを実行するソフトウェアのこと。
	　　認証サーバに対して認証を求める端末や、サーバとのやり取りの手順を実装したソフトウェアを指す
	
	●オーセンティケータ
	➡　認証器、認証子、認証符号などの意味を持つ英単語。
	　　利用者や端末などとやり取りして認証を実施するシステムや装置という意味で用いられることが多い。
	　　稀にパスワードなど認証に用いる秘密の符号を指すこともある
	
	●認証サーバ
	➡　システムの使用やネットワークへの接続を試みた人物が登録利用者本人であるかを確認する「認証」
	　　（authentication）を行うサーバ。
	　　特に、ネットワーク内でユーザー認証を一手に引き受けるサーバのことをこのように呼ぶことが多い。
	
	●多要素認証
	➡　ID・パスワードなどの「知識情報」および、「所持情報」「生体情報」という認証の3要素の中から、
	　　2つ以上の異なる認証要素を用いて認証する方法。
	　　容易に推測可能なパスワードを設定しているアカウントは未だに多く、加えて、
	　　そのパスワードを複数のサービスで使いまわしている場合は、
	　　攻撃者によってハッキングされる可能性が高まります。
	
		〇知識要素
		⇒　知識要素とは、その人の頭の中にある情報のこと
		　　パスワード、PINコード、秘密の質問　など
		　　本人しか知り得ないという前提に成り立つため、誕生日や記念日、
		　　電話番号といった個人情報から推測できる数字や覚えやすい単純な組み合わせの
		　　文字列は推奨できない
		
		〇所持要素
		⇒　所有要素とは、その人だけが持っているモノのこと
		　　スマートフォンのSMS認証やアプリ認証、トークン（ワンタイムパスワード生成端末）、
		　　ICカード　など
		　　ほかの人と共有しないことが前提で成り立ち、モノの種類によってセキュリティレベルが異なる
		　　所有要素は紛失や盗難のおそれがあるため、無効化や再発行などの手順を明確にすることが重要
		
		〇生体要素
		⇒　生体要素とはその人固有の身体的特徴のこと
		　　指紋、虹彩、顔、声紋、静脈、位置情報
		　　近年では、スマートフォンやパソコンに
		　　指紋認証や顔認証が標準で搭載されるなど普及が進んでいる
		　　指紋は、けがなどで認証情報を使用できなくなることがあるため、
		　　複数の指紋データを登録しておくことが大切
		
	●AAA
	➡　Authentication（認証）Authorization（認可）、Accounting（アカウンティング）の略称
	　　これら3つの異なるセキュリティ機能を設定するためのアーキテクチャ上の枠組み
	
		〇Authentication
		⇒　認証では、ユーザID/パスワード、デジタル証明書などの有効な
		　　クレデンシャル（信用情報）を確認し正当なユーザであるかを決定
		　　「認証」を行う際にコマンド設定で選択されたセキュリティプロトコルに応じて
		　　「チャレンジ/レスポンス、暗号化」等も提供される時がある
		
		〇Authorization
		⇒　認可では、認証に成功したユーザに対して提供する権限（ネットワークサービス）を制限する。
　		　　ユーザが発行可能なコマンドを制限したり、ネットワークのアクセス範囲を制限できる。
　		　　ユーザに認可されるタスクは、ユーザまたはグループに関連づけられるアトリビュート（属性）
　		　　により定義される。これらはローカルに設定したり、TACACS+ / RADIUSサーバで設定できる。
		
		〇Accounting
		⇒　アカウンティングでは、認証に成功したユーザに対して情報を収集する。
		　　機器にログインしたユーザ、または、ネットワークアクセスしたユーザが入力したコマンド、
　		　　接続時間、システムイベント等の情報をタイムスタンプをつけてログに記録することができる。
		　　収集した情報はRADIUSサーバなどに送信して管理して課金、監査、レポート作成に利用される。
		
	●RADIUS
	➡　Remote Authentication Dial In User Service
	　　ネットワーク上でユーザー認証を実現するプロトコル
	　　ISP（インターネットサービスプロバイダ）に広く採用
	　　最近ではテレワークを実現するために必要となるリモートアクセス環境において、
	　　VPNゲートウェイ機器などと連携し不正アクセスを防止するための認証を提供している。
